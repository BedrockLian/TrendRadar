# render_markdown.py 输出格式规范

> 简报渲染由 `render_markdown.py` 纯脚本接管（~0s，零 token）。格式硬编码，本文档是唯一权威来源。

## 输出格式

```
### Hermes日报 · YYYY-MM-DD（早报/午报/晚报）

### 📰 头条

🆕 1. **标题**
摘要（30-60字中文，英文已翻译）
[查看原文](url)【媒体名】


🔥 2. **标题**
...
```

## 格式铁律

| 规则 | 说明 |
|------|------|
| 标题行 | `### Hermes日报 · YYYY-MM-DD（早报/午报/晚报）` |
| 板块顺序 | 📰头条 → 🌏外媒看华 → 💻科技 → 📊经济民生 → 🎮游戏 |
| 全局编号 | 从头条 1 到游戏 N，按板块连续不重置 |
| 热度 emoji | 🆕首发 / 🔥跨源热点 / 🔄持续进展 / 📌深度分析 |
| 空板块 | 无条目的板块输出 `暂无内容` |

### 空行（手机端铁律）

| 位置 | 规则 |
|------|------|
| 标题 ↔ 摘要 ↔ 链接之间 | **1 空行** `\n\n` |
| 条目之间 | **2 空行** `\n\n\n` |
| 板块标题前后 | **2 空行** `\n\n\n` |

WeCom 单空行在手机上折叠成一行，必须双空行。

### 尾注
`**📋 共 N 条 · 头条A  外媒看华B  科技C  经济民生D  游戏E**` — 加粗，栏目间双空格。从 curated JSON 读实际条数生成。

### 分片规则
- 超 4000 字符按板块分片，片间 ≥1.5s 延迟
- 片首不重复标题行，尾注仅末片

### 禁止
- 横线 `---`、表格、引用、斜体、删除线（WeCom 不支持）
- 零前置文本：首字符必须是 `###`，禁止 "Now let me" 等自述

### 板块 emoji 硬编码
脚本强制使用 `DOMAIN_EMOJI`/`DOMAIN_LABEL` 中的固定值：`📰 头条` `🌏 外媒看华` `💻 科技` `📊 经济民生` `🎮 游戏`。LLM 不能用错 emoji。

## 使用

```bash
$PYTHON scripts/render_markdown.py --push-id morning 2>/dev/null
```
stdout = 完整简报，Agent 直接投递不修改。

## 自检命令

```bash
grep -ciE 'All data gathered|Now producing|Let me|Here is|I will now'  # 应为 0
grep -c '\\[查看原文\\].*【.*】'        # 超链接格式
grep -cP '\\*\\*📋 共.*  .*'           # 尾注
head -1 | grep -cP '^### Hermes日报'   # 首行
grep -cP '^[A-Z][a-z]+ [a-z]+ '        # 英文残留，应为 0
grep -c '---'                           # 横线残留，应为 0
```
