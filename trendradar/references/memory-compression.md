# 记忆压缩协议

当 MEMORY.md 或 USER.md >90% 时，Agent 手动压缩。

## 识别可压缩内容

| 类别 | 处理 |
|------|------|
| **重复事实** | 合并为一条，分号分隔 |
| **过时任务** | 删除（"已修复bug"等） |
| **版本号/数字** | 移入 skill 或删除 |
| **详细教训** | 缩至一句话 |
| **指令前缀** | 删除（"零废话"等） |
| **偏好** | 保留，精简措辞 |

## 操作

用 `memory(action='replace', target='memory', old_text=..., content=...)` 逐条替换。

**推荐流程（今日已验证）：**

1. 识别重叠条目（如"双模型混合"+"TrendRadar架构"+"性能优化器"）→ 合并为一条
2. 删除过期任务产物（v5.2.1精修完成记录等）
3. 用 `replace` 将首个条目替换为合并版（含所有关键信息）
4. 用 `remove` 删除被替换掉的其他冗余条目
5. 英文内容适合转中文压缩（50%+空间节省）
6. 重复直到使用率 <65%

每次 `replace` 会显示当前 usage%，用于判断进度。

**铁律**：
- 每次只替换一条，不批量（防误删）
- 合并用分号 `；` 分隔
- 删除前确认非当前有效信息
- 用户偏好/路径/API/配置不动
- 完成后确认使用率 <75%

## 示例

**压缩前(91%)**：
```
零废话。民生经济... TrendRadar v4.9.0，17脚本，35RSS源，544关键词。11步cron...
```

**压缩后(60%)**：
```
偏好民生经济... TrendRadar：Flash管线+Pro晚报3Agent深析。多RSS源，sources.json动态管理。
cron步骤由脚本管线内生。不展开模板变量，步骤0走push_slot_detect。
```

## 不做独立 cron

记忆不做定时 cron 压缩 — Agent 值守确保有价值信息不被误删。
