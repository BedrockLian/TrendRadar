## 🔴 P0 阻塞问题（必修，不修就断流）

### P0-1. Gateway 调度死锁 — 投递链路断了 9 天的根因

**事实**（Agent A 实证）：
- `schtasks /Query` 输出：任务**已禁用 (DISABLED)**，上次结果 1（失败），无下次运行时间
- `gateway-service/Hermes_Gateway.cmd` 目录**是空的**（被清理）
- `gateway.log` 最后一条 `2026-06-10 20:47:08`（10 天前最后一次启动）
- `tui_gateway_crash.log` 最后一条 `2026-06-17 09:00:32 Thread-11 (_readerthread) UnicodeDecodeError: 'gbk' codec can't decode byte 0x80`
- 根因：`C:\Users\ASUS\AppData\Local\hermes\hermes-agent\tui_gateway\server.py:270` 用 `for line in self.proc.stderr` 在 GBK 编码下读子进程 stdout 崩溃
- 触发器是 `LogonTrigger` + 笔记本 `DisallowStartIfOnBatteries=true`（经常电池模式）→ 即使重启也不启动

**链路影响**：
```
Gateway 死 → cron 5 个 job 无法自动触发 → push_log 6/17 morning 后断流
→ 漏推 10 slot（6/17 noon/evening, 6/18 morning/noon/evening, 6/19 morning/noon/evening, 6/20 morning/noon）
→ Agent A 手动 watchdog 补了 3 个（6/10 noon, 6/17 morning, 6/20 evening）
→ 还剩 7 个永远补不回来（archive/ 也没原料）
```

**修法**（两步，先修代码再启调度）：
1. **修 GBK 解码 bug**（最小改）：在 `hermes-agent/tui_gateway/server.py:270` 改 `for line in self.proc.stderr` 为 `io.TextIOWrapper(self.proc.stderr, encoding='utf-8', errors='replace')`，或在 Popen 时加 `encoding='utf-8', errors='replace'` 并 text 模式打开
2. **重启 Gateway + 改触发器**：
   ```cmd
   schtasks /Change /TN Hermes_Gateway /Enable
   schtasks /Change /TN Hermes_Gateway /Setting /STOPONBATTERY:NO /STARTONBATTERY:NO
   schtasks /Run /TN Hermes_Gateway
   # 长期方案：改 LogonTrigger → ONSTART，或写 schtasks 事件触发
   ```

## ⚠️ P1 隐患（强烈建议修）

### P1-1. `paths.py` 双源陷阱 — 真实存在，靠 symlink 兜底

**事实**（Agent B 验证）：
- 内外层 `paths.py` md5 完全相同（`3f5ce381a01bf404a503d4aca5dc002a`）
- 都用 `Path(__file__).resolve().parents[1]` 解析：
  - 内层 `trendradar/scripts/paths.py` → `parents[1] = trendradar/`
  - 外层 `scripts/paths.py` → `parents[1] = trendradar/`
- 但 `from trendradar.scripts.paths import TRENDRADAR_HOME` 拿到的字符串路径**指内层**
- 物理路径靠 `trendradar/data` symlink 兜底，**realpath 一致**
- **symlink 一断即静默写错目录**

**修法**（推荐方案 B）：
- `paths.py` 改用 `os.environ['TRENDRADAR_HOME']`（统一由 cron 注入），彻底去掉 `__file__` 依赖
- 已有 `file_utils.py` 用此模式，可复用
- 加 runtime assert：`assert DATA_DIR.resolve() == Path(__file__).resolve().parents[2] / 'data'`，否则 fail-fast
