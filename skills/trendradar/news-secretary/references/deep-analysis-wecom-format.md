# 深度分析 WeCom 投递格式规范（v3.0）

## 通用规则
- 每篇分析作为**独立消息**投递，不合并
- 使用标准 markdown pipe 表格 `|`，不用框线字符（企业微信支持 pipe table）
- 仅用 `###` 三级标题，不用 `#`/`##`
- 不用代码块、横线 `---`、HTML 标签
- 每条消息控制在 4KB 以内

## 三篇分析格式模板

### 📊 趋势方向
```
### 📊 趋势方向 · {YYYY-MM-DD}

| 关键指标 | 今日数据 | 趋势信号 |
|----------|----------|----------|
| {指标1} | {数据} | {信号↑/↓/→} |
| ... | ... | ... |

{emoji} **{标题}** 分析正文段落，不超过3行。

📊 **总结句：** 一句话提炼核心判断。
```

### 🔗 跨域交叉
```
### 🔗 跨域交叉 · {YYYY-MM-DD}

| 事件 | 影响领域 | 联动机制 |
|-----|-----|-----|
| {emoji} {A} → {emoji} {B} | {领域1}·{领域2} | 一句话说清联动逻辑 |
| ... | ... | ... |

**交叉一：{标题}** 分析正文。

**交叉二：{标题}** 分析正文。
```

### ⚠️ 风险预警
```
### ⚠️ 风险预警 · {YYYY-MM-DD}

| 风险项 | 等级 | 触发条件 | 影响面 |
|-----|-----|-----|-----|
| {风险名} | 🔴高 | {具体可观测条件} | {影响描述} |
| {风险名} | 🟡中 | {具体可观测条件} | {影响描述} |
| {风险名} | 🟢关注 | {具体可观测条件} | {影响描述} |

> **综合判断：** 2-3句总结。
```

## 数据源
- 趋势和跨域的数据卡来源于 `curated_{push_id}_{date}.json` 各 domain 筛选
- 风险分析基于当日精选文章蕴含的风险信号
- 三篇由 Pro 模型 `delegate_task` 并行生成

## 投递方式
每条分析格式化后通过 `render_deep_analysis.py --topic "主题" --push-id evening --context` 处理，
然后用 `cat file | hermes send --to wecom:bl` 独立发送。
